# Реалізація потокового шифру Rabbit (ISO/IEC 18033-4)

Програма є реалізацію потокового шифру **Rabbit** мовою C++ згідно з міжнародним стандартом **ISO/IEC 18033-4:2011**.

## Особливості проєкту

* **Відповідність стандарту:** Алгоритм реалізований ідентично до математичної моделі ISO, що підтверджено тестами з Annex С.4.
* **Робота з великими файлами:** Завдяки буферу об'ємом **4 МБ**, програма може шифрувати файли будь-якого розміру і без перевантаження оперативної пам'яті.
* **Архітектура:** Логіка шифрування (ядро) відділена від інтерфейсу користувача та роботи з файлами.
* **Псевдовипадкові ключі:** Реалізовано генератор ключів (24 байти: 16 КБ ключ + 8 КБ IV).
* **Тестування:** Покриття тестами (Known Answer Tests) на основі додатку Annex C4.3.
* **Кросплатформенність:** Збирання через CMake (підтримка Windows/MSVC та Linux/GCC).
* **Стійкість:** Реалізовано обробку виняткових ситуацій (відсутність прав доступу, некоректні розміри файлів ключа і тд) без аварійного завершення програми.

## Використані інструменти та технології

* **Мова:** C++23.
* **Збирання:** CMake 3.28+.
* **Тестування:** Google Test (GTest) — для автоматичної перевірки правильності обчислень.
* **CI/CD:** GitHub Actions.

## Як зібрати та запустити

### Попередні вимоги
* Компілятор з підтримкою C++23.
* Встановлений CMake.

### Приклади використання
Програма працює через командний рядок. Щоб переглянути всі доступні параметри та режими роботи, потрібно запустити програму з аргументом "-h":

```bash
./RabbitCipher -h
```

Це викличе вбудовану довідку програми, де описані всі обов'язкові та додаткові параметри.

## Тестування

Проєкт проходить 12 тестів (Known Answer Tests), які перевіряють:
1.  Відповідність гами еталонним векторам ISO (Annex C.4.2).
2.  Правильність внутрішніх станів шифру на кроках S(0)–S(4)(Annex C.4.3).
3.  Коректність обробки помилок (невірний розмір ключа, відсутність файлів).
